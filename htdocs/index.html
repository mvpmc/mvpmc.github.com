This project is just beginning. The goal is to create a suite of libraries for controlling the Hauppauge MediaMVP, as well as a media player capable of playing media files served by software such as MythTV.

If you wish to get involved, please send me email here. You should also sign up for the developer mailing list here.


dongle.bin builder

This is a shell script which automates the process of building a new dongle.bin file. It will extract the IBM kernel modules from a user supplied Hauppauge dongle.bin, and create a new dongle.bin from the filesystem and kernel supplied with this script.

The filesystem contained in this tarball contains the mvpmc media player. It is capable of playing recorded mpeg files off a MythTV server, as well as playing mpeg and mp3 files off an NFS filesystem. It can also display bmp, gif, jpg, and png images.

Building the source code

To build the code in CVS, check out the mvplib and tests CVS modules. Also, get Microwindows 0.90 here, and a MediaMVP patch for it here. Apply the patch to Microwindows, and modify src/config and src/Makefile.rules to suite your build infrastructure.

Next, run autoconf and configure in the mvplib and tests directories. You will need to use the --with-mvplib and --with-microwindows options with configure to tell it where those directories are. Then build mvplib, microwindows, and tests, in that order. This will give you some widget test programs, and a couple demo media players, which will demonstrate the use of the MVP libraries and nano-X.

Also note that the test program will produce binaries which use a shared libc. So make sure you either have a ramdisk suitable containing a shared libc, or you change the makefiles to link the test programs static.

Hopefully this will all be automated soon, but for now it is a manual process.
